HTML5 FOR WEB DESIGNERS

  • HTML5 is the cirrent version of the web mark-up language. It has is origins from HTML 2.0
  • XHTML 1 introduced a stricter markup from XML
  • XHTML 2 advanced from XHTML 1.0 and sought to replace HTML 4.01 which was a complete failure
  • WHATWG a rebellion that broke off from W3C whose main goal is to extend HTML to allow the creation of web applications.
  • Currently two groups work on HTML5.
  • HTML5 design principles are:-
    • Supporting the existing content
    • Don't reinvent the wheel
    • Pave the cowpaths
    • Priority of constituencies
  • HTML5 has specifications for what needs to be done in the event well formatted markup is parsed and likewise when badly formatted markup is parsed
  • To echo the point above HTML5 is written for both authors and implementers
  • One key change is how the doctype is defined in HTML5 ***. It is so because it HTML5 has to be backward compatible and therefore it makes no sense stating the version number
  • Browsers support features not doctype
  • You can omit the type attribute from script and link tags for javascript and stylesheets since they are the standard
  • Obsolete elements such as frame, frameset, noframes, acronym, font, big, center, strike, bgcolor, cellspacing, cellpadding and valign are not supported in HTML5 although due to backwards compatibility they can still be used.
  • smallis used for small print
  • Strongconveys extra importance
  • i is used to show an alternate voice
  • em is used to emphasis.
  • cite is for an authors works
  • a is now an anchor on steroids. It is an important tag since its the one that connects the web
  • Javascript APIs are now specified so now browser vendors have a script from which to go on. This ensures better compatibility
  • Rich media is among the new advances being brought about by HTML5
  • "Plugins" would be required to get rich media to be supported by the browser. HTML5 seeks to bring that to an end by making then native to the browser
  • The canvas element is used to create dynamic pages. It does this through an API that is manipulated using javascript.
  • A canvas is good in that it can be updated and redrawn based on users actions
  • SVG although not adopted by HTML5 is better than canvas in that its DOM can be accessed
  • JavaScript should be used as an enhancement tool by using an approach referred to as Unobtrusive JavaScript.
  • The concept of boolean attributes such as autoplay, loop whose presence on a page determines if an action will be executed or not.
  • A huge problem of relying on plugins is that they are sandboxed from the rest of the of the document and awesome things like styling cannot be performed on them.
  • We see declarative replacement of commonly done scripted operations. Use with care as some of the browsers may not already support this.
  • Placeholder - Puts temporary text in a textbox that is removed once a user starts typing
  • Autofocus allows for your cursor to be put in a input element, select once you load the page. Only used once in a document
  • Required is used to validate a form input
  • Several input types are now added into the specifications that include: search, tel, number, range, url, email, colour, date, time, month, datetime, datetime-local
  • Browsers without support for this default to text input type
  • The new types allow for control/ input suited to the type to be shown and also to vallidate the values entered
  • There is a pattern attribute that allows you to use regex to define the kind of input you require
  • Class attribute can be used to add more semantic value to elements. CSS is just an example of a technology that makes use of the class attribute during styling.
  • Microformats which are a set of agreed uon conventions by community uses the class attribute to extend HTML.
  • Microformats work based on the pareto principal
  • There are a bunch of new text-level semantics that have been introduced in HTML5.
